Native iOS - Glassmorphism

Native iOS Glassmorphism

Here I am experimenting with Glassmorphism in SwiftUI. Glassmorphism is a UI trend, but also standard that has been used lightly for years. For this experiment I decided to go extremely loud and rude with the colours and would highly recommend toning it down for a real design.

Cheerful Colors

In this design I have used two colour ramps on the background vs foreground that cover different parts of the spectrum. This is about as loud as one can get. Here I am using warm colours for the background gradient and cool colours for the bar chart, foreground icons and foreground text.

Accessibility

Glassmorphism is more accessible than Neumorphism, but not without its challenges. A glossy UI can take the user's attention away from the text so a lot of care must be taken when using it excessively, especially when the UI gets more complex.

Design System

I made the decision to squeeze three UI styles, each having three layout, inside one app. The reason for this is because I want to create a living design system. The demonstrator is populated with a data model and dummy data and the user can add and delete information and this is persisted. Rather than keep separate branches, I wanted to keep all the design styles in one place while debugging and setting up the core data stack.

Loud but fun

Here I use cool colors for the background gradient and warm colours for the line charts, foreground icons and foreground text. Menu items for example can be a great opportunity to enjoy glassmorphism whereas the reader probably shouldn't be reading long passages this way.

Contrast challenges

Contrast is still very important. The contrast between the dark grey text and its background and the contrast between the purple text/icons and background can be improved even further.

Can always water it down

With simple user interfaces one can have a lot of fun with glassmorphism, especially when changing backgrounds. One thing I haven’t tried yet is using a lighter pastel gradient background. This could make the UI less intense and even more readable.

A Clean and Organised Design System

I plan to spend some time cleaning up and simplifying the Design System code, to ensure consistency and also to make it even easier to reuse components.

Atmosphere

As we know, different colours create different moods so I am very keen to enable the user to change or customize the UI easily

What's next

Once this design system is finished I am planning to go deeper into the data visualisations and hopefully have a large library of charts with different UI styles.
