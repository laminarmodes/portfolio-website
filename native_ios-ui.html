Neumorphism

Neumorphism was an interesting concept that hasn't quite gotten much traction yet. It is a new generation of skeuomorphism, but rather than making components resemble real objects, they are kept simple and minimalistic, but extend out in 3D.

Implementation

I wanted to test the aesthetic feel of neomorphism in high fidelity so I made a demonstrator using SwiftUI. I converted a flat design into a neuromorphic UI in it's most simple form through a status indicator, button and various card layouts.

Accessibility issues

Neumorphism has been challenged, and for good reason, due to issues with accessibility.

This can be slightly overcome by incorporating the use of color around components. And while color neumorphic components can be more accessible, the slight drawback is that they become more complex to build from scratch and clutter up the code. Also coding 3D components requires experimentation, patience and a very good eye.

Transitions are also extremly important. This is something that can be very tricky to get right with 3D buttons.

If not designed well, interacting with a neumorphic button can feel creepy and unrealistic.

Element spacing

Another thing to watch out for is element spacing. The 3D effect is created with layers of shadows and so extra margin between components in needed in order to avoid cutting these shadows off. This limits the amount of components that can fit into the screen space (which actually isn't a bad thing at all).

Here, the title for each row of cards needed to be lifted away from the top of the cards to avoid cutting through the top shadow. Also, each row/title set needed additional separation to 1) protect the bottom shadows of the cards and 2) compensate for the loss in proximity between the cards and the row title.

The positives

Looking past the accessibility issues, there are still many great things that come out of this style

  • The 3D effect really brings the screen to life in a new and engaging way.
  • The extra affordance lets you go softer on the colors (though it is still good too keep them to improve accessibility).
  • It could be a great bridge into 3D design systems for augmented reality productivity apps. Here I am imagining virtual post-it notes.

Give it a go

Aside from accessibility, implementation complexity and cluttering up code (for now), neumorphism can create some very stunning user interface.

There is no harm in building a fun app with this style, but it might be a good idea to give users the option to toggle back to a more traditional interface. This can be done with a clean design system architecture.
