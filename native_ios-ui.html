Neumorphism

Neumorphism was an interesting design trends that is super cool, but hasn't quite taken off yet. It is a new generation of skeuomorphism, but rather than making components resemble real objects, they are kept simple and minimalistic, but protrude in 3D.

Implementation

I wanted to test the aesthetic feel of neomorphism in high fidelity so I created a demonstrator using SwiftUI. I converted a flat design into a neuromorphic one, in it's most simple form through a status indicator and various card layouts.

Accessibility issues

Neumorphism is challenged by issues due to accessibility.

This can be slightly overcome by incorporating the use of color around components. Color neumorphic components can be very accessibly but the slight drawback is that they become more complex to build and require patience and a good eye.

Element spacing

Another thing to watch out for is element spacing. The 3D effect is created with shadows and so extra margin between compoments in needed in order to avoid cutting off the shadow.

But it looks great

There are of course many great things that come out of this

  • The 3D effect really brings the screen to life
  • If accessibility weren't an issue, it lets you go softer on the colors while keeping affordance.
  • It is a great bridge into 3D design for augmented reality.

Give it a go

Aside from accessibility and implementation complexity, neumorphism can create a very stunning interface. There is no harm in building a fun app with it, but it might be a good idea to give users the option to toggle back to a more accessible interface. This can be done with a clean design system architecture.
