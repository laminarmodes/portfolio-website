Data Visualization

Dashboard design in Adobe XD

Information visualization and dashboard design has been a very important topic for digital products. Converting data insights into a story that can be understood by everybody is extremly challenging.

If only we could just collect requirements, generate charts, place them in a dashboard and hope for the best. Sometimes this works for simple insights, but for more complex dashboards, we need so much more.

There are many details we should pay attention to, things that we often take for granted:

  • Color matters. Chart elements of the same color will be associated with one another.
  • Location and order matter. If we intend for an element to be viewed first, then it cannot go on the bottom right side of the screen.
  • High-level summaries are extremly important, even if just a single number. Often times people use a dashboard and yet go to the back-end, or export data because they still need to compute an overall quantity. This extra work can be avoided with high-level summaries.

Online Banking UI design in Figma

The next visualisation is a responsive UI for an online banking app. The theme here is fun, trustworthy and playful and intended to create a delightful and secure banking experience for the user.

Life can be hard. Banking can be stresful. For anybody.

How can we help users, regardless of their financial situation, have a pleasant and stress-free banking experience?

How can we encourage users to use banking tools to monitor their progress and achieve their goals, even if they only have a couple hundred dollars in their account?

When it comes to dashboards, aesthetics stil matter. Here, the structure of the logo represents a purple coral reef and makes up the primary purple color of the user interface. The inner parts of the logo represent colorful fish living inside a coral reef and this is seen in the secondary colors that represent different types of accounts.

iPhone - Don't want to miss out

There is quite a lot of information to view so rather than cramming everything onto a single screen, the user is encouraged to scroll horizontally to see weekly or monthly trends and information about different accounts. A desktop-first approach was taken in this design and it will be seen later than the desktop layout is almost exactly the same, only with everything visible on the screen less need for scrolling.

I also made a point to make sure that all features (or thingies!) in the desktop app were included in the mobile version, so that the user was not "missing out" when using the mobile app.

This is really imporant.

Unexpected situations may arise where somebody is away from home and needs to do something urgent and serious with their mobile banking. Excluding features in the mobile banking could cause a lot of dissapointment and anxiety.

iPad - Careful when multi-tasking

The iPad navigation can be the same as the iPhone or alternatively placed to the right or left side for easier reaching while multi-tasking. The bottom navigation encourages holding the iPad from the bottom which presents a lot of accessibility issues for anybody holdindg the device with one hand.

It is easier to lose grip of and drop an iPad.

The navigation can be implemented to automatically adjust to the device, and this can be done within the same parts of the code using conditional statements. Even more advanced layout changes are possible such switching between a navigation drawer on an iPhone to an always-visible vertical scrolling on the iPad.

Desktop

The mobile designs, for now, are almost exactly the same as desktop. This ensures that the user doesn't get confused when switching between desktop and mobile however it is still worth exploring more efficient layouts that are optimised for each device. Also for some of the specific design differences such as the side navigation bar, it was taken into account that SwiftUI allows for the same codebase to be used for Native Mac apps and iPad apps (as well as iPhone).

There is one more thing though!

Screen privacy can be more challenging when using a laptop or desktop. This brought me to think of an interesting feature that could be added, not just to desktop but for all devices. The ability to blur or hide numbers would really enhance an online banking experience. Often we may want to log in with family, friends or colleagues present and having all the information visibile on the screen can be ackward and make a user feel anxious.

So when thinking about this interface, I am keen to add a little "eye" icon that opens and closes when the user wants to show or hide the numbers in their banking site. More thought will need to go into this though because it must be done carefully in such a way that they don't find them selves blindly performing the wrong transactions.

Final Thoughts

There are still quite a few improvements that need to be made

  • The tab icons are too large. Fortunately this actually doesn't need to be specified during the developer handover because software UI toolkits automatically calculate the correct tab icon size and spacing.
  • The text alignment could still use a little improvement.
  • The iPad and iPhone layouts could be made a little smarter.
  • The top navigation on the iPad and iPhone is cramped.
  • There is no dark theme design.
