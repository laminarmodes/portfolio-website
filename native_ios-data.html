Mobile-first data visualization

Learning Swift was a great experience and I was very happy to transition to SwiftUI and continue to explore one of my favorite parts of coding which is data visualization. Taking a mobile-first design approach to data visualization is something I have always been interested in because it could really free the analyst from their desk and allow them to explore data, monitor trends, do comparisons and listen out for alerts while moving around.

Ideas -> How does it work?

After coming up with any concept, I always try to break it down into the basic steps in order to identify gaps in the concept and to ensure it is realistic. I know what the user is going to accomplish but just need to illustrate how they will go about it. Here I had the idea of the user wanting to do three things 1) add information quickly, 2) sort information comfortably and 3) present information clearly.

What will it look like?

People need diverse options to organize, communicate & interpret information: charts, words, symbols, images, tables, lists. I wanted to really brainstorm different ways to toggle between all options in a simple UI. The mobile-first approach though comes limits in real estate for rendering data so smart design patterns are needed to help simplify the user interface, without losing information.

SwiftUI

It's wonderful to experience how Data Binding is super simple to implement in SwiftUI, allowing more focus on creating a complex UI that lets the user explore, drill-down, compare, scan and browse. The actual user input (shown later) is persisted, but in a simple way for now in order to collect user feedback and iterate on.

Data Summary

Color-blind friendly color palettes can be used to label a list of data sets. In the bar chart, each bar height represents the number of items that are contained inside each large summary card. The color of the bar then corresponds to the color of the large summary card.

An additional interaction could be, when the user selects a bar, it automatically scrolls to the corresponding card for them.

Casual scrolling

This view would allow the user to drill down past the summary, and look at all the information contained within all summary cards at once. Each row should correspond to the color of the large card.

Each 'row of cards' corresponds to a single line on the line graph and each 'individual card' corresponds to a 'single point' on the line. Each project can have a different overall color theme; in this case, a warm color-blind friendly color ramp.

Preserving details

Clicking that card will reveal the details of the card and alternatively, the user can view a row of cards in a list view which may be preferred for presentations and discussions.

User input

New cards can be added to each individual row (see 'Test Item, Test Description') and a new category can be added as an empty list (see 'Test List').

Transition animations

However when a new card is added, the transition can be sudden and the card may go unnoticed. It is important to remember to use animations for these interactions.

What's next

I haven't actually revealed what this app is for because it is still work in progress but was keen to share some features of the user interface.

There are a lot more features that could be added, for instance, the ability to drag cards to either reorder them, more move them to a new category. These things though will be addressed after more user testing is done.
